Chapter One -- The Beginning

The Problem

LLM inference was broken. Traditional load balancers treated every GPU pod as interchangeable, randomly spraying requests across the fleet. But LLMs carry enormous context in KV caches. The right pod can serve 10x faster than the wrong one.

Organizations had two options: build bespoke infrastructure (expensive and fragile) or accept terrible GPU utilization. The Kubernetes ecosystem had no answer.

🎯
Random Routing
Requests scattered across pods with no awareness of which GPU already held relevant KV cache data.
🔥
Wasted GPU Cycles
Cold starts on every request. Expensive prefill recomputed again and again across the cluster.
🔒
Vendor Lock-in
Proprietary inference platforms offered optimization but at the cost of flexibility and portability.
⚙️
No K8s Answer
The cloud-native ecosystem had no solution for intelligent, GPU-aware LLM inference scheduling.

What Makes It Different

Production-Grade Features

🧠
Intelligent Routing
The Endpoint Picker (EPP) scores every pod by cache affinity, queue depth, and active load. Requests land on the GPU most likely to serve them instantly. The result: 80%+ cache hit rates, slashing time-to-first-token by orders of magnitude.
⚡
Prefill/Decode Disaggregation
Separate GPU pools for prompt processing (prefill) and token generation (decode). Each phase scales independently based on workload mix, connected by high-speed NIXL interconnect for KV cache transfer.
🚀
Scale-to-Zero & HA
Active-active high availability ensures no single point of failure. Scale-to-zero autoscaling releases idle GPUs automatically. Hierarchical KV cache offloading preserves context across scale events.
